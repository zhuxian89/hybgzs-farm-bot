## 根本原因

### 正则表达式过于严格

**位置1**: `clickPlantConfirm()` (行 703)
```javascript
// ❌ 错误：要求精确匹配，不能有其他内容
/^种植 x\\d+$/.test(value)

// 实际按钮文字
"种植 x6（需购买 6 个）"

// 匹配结果：false ❌
```

**位置2**: `readPlantDialogState()` (行 663)
```javascript
// ❌ 错误：同样要求精确匹配
/^种植 x\\d+$/.test(text)
```

### 为什么匹配失败

正则 `/^种植 x\\d+$/` 的含义：
- `^` - 字符串开始
- `种植 x\\d+` - 必须是"种植 x数字"
- `$` - 字符串结束（**不能有任何其他字符**）

实际按钮文字：`种植 x6（需购买 6 个）`
- 有额外的 `（需购买 6 个）` → 不匹配 ❌

## 修复方案

### 修改正则为前缀匹配

**位置1**: `clickPlantConfirm()` (行 703)
```javascript
// ✅ 正确：只要以"种植 x数字"开头即可
/^种植\\s*x\\d+/.test(value)

// 现在可以匹配：
// - "种植 x6"
// - "种植 x6（需购买 6 个）"
// - "种植x6"（允许空格可选）
```

**位置2**: `readPlantDialogState()` (行 663)
```javascript
// ✅ 正确：同样改为前缀匹配
/^种植\\s*x\\d+/.test(text)
```

### 其他相关位置

**位置3**: `isCropSelectedForPlanting()` (行 741) - 已经是宽松的
```javascript
// 这个已经是前缀匹配，不需要修改
/^种植\\s*x?\\d+/.test(text)
```
