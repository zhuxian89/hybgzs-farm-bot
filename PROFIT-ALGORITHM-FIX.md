# 收益算法优化 - 改为按24小时总收益排序

## 🐛 问题描述

### 原来的算法（错误）
按 **每小时利润** 排序选择作物：
```javascript
.sort((a, b) => b.profitPerHour - a.profitPerHour);
```

### 为什么错了？

**每小时利润 ≠ 24小时总收益**

举例：
- 胡萝卜：1.2 金币/小时，但 0.5 小时一轮 → 48 轮/天 = 173 金币/天
- 草莓：1.32 金币/小时，但 4 小时一轮 → 6 轮/天 = 189 金币/天

如果只看每小时利润，草莓（1.32）比胡萝卜（1.2）高，会选草莓。
但实际 24 小时总收益：草莓 189 > 胡萝卜 173，这次选对了。

**但如果价格波动，可能选错！**

比如某天番茄价格暴涨：
- 番茄每小时 2.0 金币/小时
- 南瓜每小时 1.57 金币/小时

按每小时算，选番茄。但如果番茄 24 小时总收益反而更低，就亏了！

## ✅ 修复方案

### 改为按 **24小时总收益** 排序

**位置**: `scripts/farm-bot.js` 第 90-112 行

**修改**:
```javascript
function rankCropsByProfit(crops, exchangePrices) {
  return crops
    .map((crop) => {
      const sellPrice = exchangePrices[crop.name];
      if (!Number.isFinite(sellPrice)) return null;
      const revenue = crop.yield * sellPrice;
      const profit = revenue - crop.seedPrice;
      const profitPerHour = profit / crop.growHours;
      const roundsPerDay = 24 / crop.growHours;              // 👈 新增：计算每天轮数
      const dailyProfit = profit * roundsPerDay;            // 👈 新增：计算每天总收益
      return {
        ...crop,
        sellPrice,
        costPrice: crop.seedPrice,
        revenue,
        profit,
        profitPerHour,
        roundsPerDay,                                        // 👈 新增
        dailyProfit                                          // 👈 新增
      };
    })
    .filter(Boolean)
    .sort((a, b) => b.dailyProfit - a.dailyProfit);        // 👈 改：按每天总收益排序
}
```

### 更新序列化和日志输出

1. **serializeRanking** (行 156-167) - 保存 `dailyProfit`
2. **describeStrategy** (行 198-206) - 显示 "XX金币/天" 而不是 "XX金币/h"
3. **策略重算日志** (行 1553) - 显示 "XX金币/天"
