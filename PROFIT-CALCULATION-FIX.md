# 利润计算算法修复

## 🐛 问题描述

### 旧算法（错误）

**假设每轮都要花钱买种子**：
```javascript
const profit = revenue - crop.seedPrice;  // ❌ 每轮都减种子成本
const dailyProfit = profit * roundsPerDay;
```

**计算结果**（10块地，番茄）：
- 每轮收入：5 × 0.71 = 3.55 金币
- 每轮成本：2 金币（种子）
- 每轮利润：3.55 - 2 = 1.55 金币
- 每天利润：1.55 × 24 = **37.2 金币** ❌

但实际上：
- ❌ 忽略了**留种机制**
- ❌ 种子来自上一轮收获，不需要额外花钱
- ❌ 实际收益应该是 **681.6 金币/天**，算少了 **18 倍**！

## ✅ 新算法（正确）

### 核心逻辑

**种子来自上一轮收获，只卖多余的！**

```javascript
// N块地的情况
const harvestPerRound = crop.yield * totalSlots;  // 每轮收获
const sellPerRound = harvestPerRound - totalSlots; // 卖出（保留种子）
const incomePerHour = sellPerRound * sellPrice;    // 每小时收入
const dailyIncome = incomePerHour * roundsPerDay;  // 每天收入
```

### 计算示例（10块地，番茄）

```
每轮收获 = 5个/块 × 10块 = 50个番茄
保留种子 = 10个（下轮种植）
卖出数量 = 50 - 10 = 40个
每小时收入 = 40 × 0.71 = 28.4 金币
每天收入 = 28.4 × 24 = 681.6 金币 ✅
```
